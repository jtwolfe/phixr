## Why This Was Archived

The previous documentation reflected an earlier understanding of the project that has been superseded by the clarified requirements:

- Two-mode system (Independent vs Vibe)
- Natural language GitLab interaction priority
- OpenCode WebUI preservation (not rebuilding)
- Specific branch and Git workflow requirements
- Shared visibility vs true multi-user editing
